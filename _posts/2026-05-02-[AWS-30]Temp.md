🗺️ 지도 개선 요구사항 정리
1️⃣ 클러스터 색상 기준 변경
기존:
마커 개수 기준 (기본 MarkerCluster)
변경:
해당 클러스터에 포함된 집들의 평균 가격 기준

👉 의미:

“많은 지역”이 아니라 “비싼 지역”을 직관적으로 보여줌

2️⃣ 가격 구간 기반 색상 그라데이션
구간:
~ 400k
400k ~ 500k
500k ~ 600k
600k ~ 700k
700k ~ 800k
800k ~ 900k
900k ~ 1M
1M+
색상:
초록 → 연두 → 노랑 → 주황 → 빨강

👉 의미:

가격이 올라갈수록 자연스럽게 색이 변화

3️⃣ 클러스터 내부 정보 표시
표시 내용:
평균 가격 (예: $720k)
포함된 집 개수 (예: 12 homes)

👉 의미:

단순 점이 아니라 요약 정보 제공

4️⃣ 클러스터 영역 시각화
기존:
마우스 올려야 범위 보임
변경:
항상 영역 표시 (rectangle)

👉 특징:

클러스터가 포함하는 영역을 지도에 직접 그림
색상 = 해당 영역 평균 가격 기준

👉 의미:

“이 구역 전체가 어떤 가격대인지” 직관적으로 이해 가능

5️⃣ 가독성 개선 (중요 UX)
문제:
노란색 위에 흰 글자 안 보임
해결:
배경 밝기에 따라 텍스트 색 자동 변경
밝은 배경 → 검정 글자
어두운 배경 → 흰 글자

👉 의미:

모든 색상에서 항상 읽히도록 개선

6️⃣ 성능 최적화
설정:
removeOutsideVisibleBounds: true

👉 의미:

현재 화면 밖 마커 제거
렌더링 성능 개선
7️⃣ 줌 레벨 기반 동작
줌인:
클러스터 → 개별 집으로 분해
줌아웃:
다시 클러스터링

👉 의미:

데이터 밀도에 맞게 자동 시각화

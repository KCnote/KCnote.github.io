## <b>1. RDS</b>

!["aws-rds0"](/assets/img/develop/aws-rds0.png)

Amazon RDS is a managed relational database service that allows you to run databases without managing infrastructure.

AWS automatically handles:

* OS patching
* Backup
* Failover
* Monitoring

#### <b>1-1. Supported Engines</b>

* MySQL
* PostgreSQL
* MariaDB
* Oracle
* SQL Server
* Amazon Aurora

#### <b>1-2. Architecture (with EC2)</b>

```
User
→ ALB
→ EC2 (Application Server)
→ RDS (Database)
```

👉 Users do NOT access RDS directly
👉 Only EC2 communicates with RDS

#### <b>1-3. Network Placement</b>

```
Public Subnet:
- ALB

Private Subnet:
- EC2
- RDS
```

👉 RDS must be placed in a **Private Subnet**

#### <b>1-4. Performance Factors</b>

* Instance size (CPU / RAM)
* Storage IOPS
* Number of connections
* Query complexity

## <b>2. How to work RDS</b>

| Component      | Role                  |
| -------------- | --------------------- |
| RDS            | Managed database      |
| Endpoint       | Database access point |
| Security Group | Access control        |
| Multi-AZ       | High availability     |
| Read Replica   | Read scaling          |

#### <b>2-1. Connectivity Flow</b>

##### Application → Database

```
EC2 → RDS Endpoint
```

Example:

```
mydb.abc123.ap-southeast-2.rds.amazonaws.com:3306
```

##### Protocol

* MySQL → TCP 3306
* PostgreSQL → TCP 5432

#### <b>2-2. Routing (VPC Internal)</b>

```
10.0.0.0/16 → local
```

👉 Communication stays inside VPC

#### <b>2-3. Security</b>

##### RDS Security Group

```
Inbound:
Port: 3306
Source: EC2 Security Group
```

```
Outbound:
Allow all (default)
```

```
EC2 → RDS ✅
Internet → RDS ❌
```

#### <b>2-4. Multi-AZ</b>

##### Single AZ

```
EC2 → RDS (Primary)
```

##### Multi-AZ

```
Primary DB (AZ1)
↓ replication
Standby DB (AZ2)
```

##### Failover

```
Primary failure
→ Automatic switch to standby
```

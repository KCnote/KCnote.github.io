## <b>1. S3</b>

!["aws-s3-01"](/assets/img/develop/aws-s3-01.png)

**Amazon S3 (Simple Storage Service)** is an object storage service that provides:

* High durability (99.999999999%)
* Scalability (virtually unlimited)
* High availability
* Simple API-based access

👉 In short:

> **S3 is a storage for files (objects), not servers**

#### <b>1-1. Bucket</b>

* Container for storing objects
* Must have globally unique name

👉 Example:

```
my-app-images
my-backup-data
```

#### <b>1-2. Object</b>

* Actual data stored in S3
* Consists of:

  * File (data)
  * Key (path)
  * Metadata

👉 Example:

```
Key: images/profile.png
```

#### <b>1-3. Key (Path)</b>

S3 doesn’t have real folders
👉 it uses **key as virtual path**

```
bucket
 └── images/
      └── profile.png
```

👉 Actually:

```
images/profile.png
```

#### <b>1-4. Important Characteristics</b>

##### Durability

* 99.999999999% (11 nines)

##### Scalability

* Unlimited storage

##### No server management

👉 No EC2 required

## <b>2. How to work S3</b>

```
Client (App / Browser)
        ↓
      Internet
        ↓
      S3 Bucket
        ↓
    Object Storage
```

1. Upload file → S3
2. S3 stores as object
3. Access via:

   * URL
   * SDK (C++, Python, etc.)

```
EC2 / Backend
    ↓
 Download from S3
    ↓
 Load into Memory
    ↓
 Process
```

#### <b>2-1. Security</b>

##### IAM Policy

* Controls access via AWS IAM

##### Bucket Policy

* Controls public/private access

##### ACL (Legacy)

* Old method (not recommended)

#### <b>2-2. Real Use Cases</b>

##### Static Website Hosting

* HTML / CSS / JS hosting
* Often combined with CloudFront (CDN)

##### Image / Video Storage

* Profile images
* Media files

##### Backup & Logs

* DB backup
* Server logs

##### Data Pipeline

* Used with Lambda / Athena

#### <b>2-3. Storage Classes</b>

| Class                | Use Case            |
| -------------------- | ------------------- |
| Standard             | Frequently accessed |
| Intelligent-Tiering  | Auto optimize cost  |
| Standard-IA          | Infrequent access   |
| Glacier              | Archive             |
| Glacier Deep Archive | Long-term backup    |

#### <b>3-5. Step 2. Security</b>

!["aws-s3-05"](/assets/img/develop/aws-s3-05.png)

1. Object Ownership: Controls who owns objects in the bucket

    - Options:
        - Bucket owner enforced (Recommended)
            → ACLs disabled, bucket owner owns everything
        - ACLs enabled
            → Object uploader can own objects

2. Block Public Access: Prevents public access to bucket and objects

    - Options:
        - Block all public access (Recommended)
            → No public access allowed

3. Bucket Versioning: Keeps multiple versions of the same object

    - Options:
        - Disable → Only latest version exists
        - Enable → Stores all versions

## <b>4. S3 vs EBS vs EFS</b>

| Feature       | S3           | EBS     | EFS            |
| ------------- | ------------ | ------- | -------------- |
| Type          | Object       | Block   | File           |
| Attach to EC2 | ❌            | ✅       | ✅              |
| Access        | HTTP         | OS disk | Network FS     |
| Use case      | File storage | OS / DB | Shared storage |
